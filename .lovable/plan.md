

# Save Wedding Brand Identity as Reference Document

## What
Store the complete Brand Identity North Star document as a persistent project file that serves as the authoritative reference whenever building or modifying the wedding page.

## Where
Create a single file at `.lovable/wedding-brand-identity.md` --- adjacent to the existing `.lovable/plan.md`, keeping all project governance documents in one place.

## Contents
The file will contain the exact brand identity document you provided, formatted as clean Markdown with all 12 sections preserved verbatim:

1. Brand Truth Table
2. Non-Negotiables, Flex, and Dealbreakers
3. Brand Spine
4. Positioning and Category Ownership
5. Customer Identity Mirror
6. Brand Story System
7. Messaging Pillars
8. Verbal Identity
9. Visual Identity Direction
10. Brand Governance
11. What Not to Do
12. Brand Archetype

## How it will be used
- Before any wedding page edit, this document is the source of truth for copy, design, voice, and guardrails
- No code changes in this step --- only the reference document is created
